4. App Store Distribution Strategy

Three-channel distribution: Web PWA first (immediate), Google Play second (1 week), Apple App Store third (2-3 weeks). The web PWA is the primary product — the app store listings are discovery channels, not the product itself.

🌐 Channel 1: Web PWA (dieseleye.app) FREE — LIVE DAY 1

Already built. Users visit dieseleye.app → install as PWA → works like a native app. Web Bluetooth works on Chrome Android. On iOS Safari, BLE is limited — users get the full dashboard experience but need the native wrapper for BLE. This is fine for demo mode and simulator.

No app store fees. No review process. Update instantly. This is your primary product forever.

📦 Channel 2: Google Play Store (TWA) $25 ONE-TIME

Method: Use PWABuilder or Bubblewrap to wrap the PWA as a Trusted Web Activity (TWA). This uses Chrome's engine — identical to the web experience. Web Bluetooth works natively on Android Chrome.

Steps: Run PWABuilder (pwabuilder.com) → enter dieseleye.app → generate Android package → upload to Google Play Console → review takes hours, not days.

Listing optimization: Title: "DieselEye — EcoDiesel Engine Monitor" / Category: Auto & Vehicles / Screenshots of live gauges on Android phone / Keywords: OBD2, diesel, DPF, EGT, Jeep, EcoDiesel, engine monitor

🍎 Channel 3: Apple App Store (Capacitor) $99/YEAR

Method: Wrap with Capacitor (capacitorjs.com). This gives you a native iOS shell with access to the @capacitor-community/bluetooth-le plugin — solving the Web Bluetooth limitation on iOS Safari. This is the KEY reason to have an App Store build.

Steps: npm install @capacitor/core @capacitor/cli → npx cap init → npx cap add ios → configure BLE plugin → build in Xcode → submit to App Store.

Apple Review Tips: Don't submit a thin PWA wrapper — Apple rejects those (Guideline 4.2). The Capacitor build with native BLE integration adds genuine native functionality. Include the BLE connection flow, offline caching, and AI chat as differentiators from "just a website."

Risk: Apple review takes 1-7 days per submission. Budget 2-3 rounds of review. Add NSBluetoothAlwaysUsageDescription to Info.plist. Test on real iPhone device (BLE doesn't work in simulator).

10. Risk Mitigation

⚠ Apple Rejection MEDIUM

Apple may reject the Capacitor wrapper as "not enough native functionality." Mitigation: The BLE integration via @capacitor-community/bluetooth-le IS native. Add native push notifications via Capacitor. Include a privacy nutrition label. Budget 2-3 review cycles.

⚠ Small Market LOW

~30-50K JL/JT EcoDiesels is small. Mitigation: Expand to RAM 1500 EcoDiesel (250K+ vehicles) in Phase 5. Same engine, same PIDs, same AI. Then Power Stroke, Duramax. The architecture is engine-agnostic by design.

⚠ Banks Makes an App MEDIUM

Banks Power could build a companion app. Mitigation: Our moat is the AI + community network effect. Banks doesn't have fleet data, anomaly detection, or Hive Mind. Position as complementary: "Works great WITH your Banks Derringer."

⚠ Solo Founder Burnout HIGH

You're running sales AND building DieselEye. Mitigation: Vibe-code with Claude Code. Automate everything possible. Set hard time limits (10 hrs/week on DieselEye). Don't over-commit to features. Ship small, ship often. Revenue from your AE job funds the runway.
