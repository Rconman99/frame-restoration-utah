# Innovate Build Handoff Template

Use this at the start of any current, legacy, or greenfield build session.

```text
Use $innovate-research in Operating Loop Mode and apply local contractor growth rules.

Build name: <name>
Mode: <current | legacy | greenfield>
Client/trade: <client and trade>
Cities: <primary cities>
Services/products: <primary services>
Goal for this session: <goal>

Read:
- AGENTS.md
- build-intelligence/README.md
- build-intelligence/build-intelligence-loop.md
- build-intelligence/implementation-queue.md
- build-intelligence/contractor-growth-category-knowledge-map.md
- build-intelligence/build_intelligence.json

Loop:
research -> categorize -> investigate -> audit -> fix -> re-audit -> implement -> measure

Instructions:
- Pick the highest-value improvement for this build.
- If there are multiple possible improvements, use Recommendation Pipeline Mode:
  source -> hydrate -> filter -> score -> select -> side effect -> feedback.
- If this is broad, high-stakes, competitive, or system-building, use World-Class Ranking Mode:
  candidate registry -> evidence feature store -> bias/diversity gates -> semantic recall -> eval harness -> feedback ledger.
- Map it to one loop stage and one artifact.
- If customer input is missing, create/update a customer request row.
- If visual quality or conversion design is weak, run the Auto-Enhance Design Lane.
- Patch the smallest useful change.
- Run relevant checks.
- Report changed files/artifacts, validation, blockers, and reusable lessons.

Safety:
- Do not publish, send, outreach, edit GBP, change DNS, run ads, add tracking scripts, or use public claims/photos/reviews/credentials without approval.
- Do not invent services, service areas, licenses, warranties, financing, emergency claims, or years in business.
- Client-approved truth beats trend research.

Auto-Enhance Design Lane:
baseline screenshot -> design/category audit -> enhancement plan -> bounded visual patch -> browser QA -> score -> reusable pattern

Recommendation Pipeline scoring:
- business value
- confidence
- speed gain
- reuse across builds
- lead/authority/design value
- learning value
- implementation cost
- compliance risk
- hype risk
- customer blocker risk
- maintenance cost
- source/recency bias risk

World-Class Ranking gates:
- every candidate has source/date/confidence/artifact/check/measure
- evidence is append-only and stale records are marked
- top actions pass vendor/popularity/recency/repeated-source/category-collapse bias checks
- semantic recall searches prior docs/templates/skills/handoffs/validators before new artifacts
- broad strategy, implementation-loop, and compliance-sensitive eval cases are defined
- shipped actions record feedback and recurring wins promote into templates, skills, validators, MCP resources, AGENTS.md, or dashboard metrics

Design gates:
- desktop/mobile render cleanly
- CTA obvious
- text fits
- no overlap
- claims approved or draft-only
- forms/calls still work
- SEO/AEO content and schema remain consistent
```
